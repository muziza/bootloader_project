# Python UART uploader для STM32H7

## Назначение
Скрипт общается с bootloader’ом в МК по UART: стирает область приложения, заливает бинарник, при желании читает его обратно для проверки и запускает приложение.

## Константы
- `CMD_ERASE`, `CMD_WRITE`, `CMD_READ`, `CMD_JUMP` — коды команд протокола.
- `ACK_OK`, `ACK_ERROR` — ответы bootloader’а.
- `BLOCK_SIZE = 32` — размер блока, должен совпадать с прошивкой bootloader’а.

## u32le(value)
Кодирует `uint32` в 4 байта little‑endian. Используется для отправки `offset` и `length`.

## wait_ack(ser, stage)
Читает 1 байт ответа от МК и проверяет его: `ACK_OK` — успех, `ACK_ERROR` — исключение, пустой ответ или другой байт — исключение с пояснением.

## erase(ser)
Печатает сообщение, отправляет `CMD_ERASE` и ждёт подтверждение через `wait_ack`.

## write_firmware(ser, firmware)
Отправляет заголовок `CMD_WRITE + offset=0 + length`, затем шлёт прошивку блоками по 32 байта, дополняя последний блок `0xFF`. В конце ждёт ACK от bootloader’а.

## read_firmware(ser, size)
Отправляет `CMD_READ + offset=0 + size`, ждёт ACK, затем читает `size` байт и проверяет, что их достаточно. Возвращает прочитанные данные.

## jump(ser)
Отправляет `CMD_JUMP` и ждёт ACK. Bootloader после этого прыгает в пользовательское приложение.

## main()
1. Разбирает аргументы: путь к `.bin`, порт, скорость, флаги `--no-verify`, `--no-jump`.
2. Читает прошивку из файла и проверяет, что она не пустая.
3. Открывает UART (`serial.Serial`), делает паузу и очищает входной буфер.
4. Вызывает `erase()` и `write_firmware()`.
5. Если не указан `--no-verify`, читает прошивку обратно через `read_firmware()` и сравнивает с файлом.
6. Если не указан `--no-jump`, отправляет `jump()`.
7. При любой ошибке печатает сообщение на stderr и завершает работу с кодом 1.
